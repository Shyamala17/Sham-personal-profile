Linux Practice Project
Overview
This project documents a hands-on practice session for basic Linux system navigation, file management, permissions, and monitoring. The goal is to help interns become confident in managing Linux systems.

Tasks Completed
Directory Navigation

Explored filesystem using pwd, ls, and cd.
Understood difference between root (/) and home (/home/<user>).
File & Directory Management

Created directories with mkdir.
Created files with touch.
Removed files safely using rm and directories using rmdir.
File Viewing & Editing

Viewed files with cat and less.
Edited files using nano and vim.
Permissions & Ownership

Checked permissions with ls -l.
Modified permissions using chmod.
Changed ownership using chown.
System Monitoring

Monitored processes with top and htop.
Checked disk usage with df -h.
Checked memory usage with free -m.
Documentation

Recorded all commands and outputs in a linux_practice.md file.
Captured screenshots of terminal usage.
Summarized tasks and learnings in this README.
Deliverables
linux_commands.txt or linux_practice.md (command log with outputs).
Screenshots of terminal usage.
This README.md summary.
Outcome
By completing these exercises, the intern gains confidence in:

Navigating the Linux filesystem.
Managing files and directories.
Understanding permissions and ownership.
Monitoring system performance.
Documenting work professionally.
